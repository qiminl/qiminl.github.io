Fixing N Queens Problem

LP/CP Programming Contest

Problem

The N-queens problem is probably one of the most famous problems in computer science. Given an NxN chess board and N queens, the goal of the N-queens problem is to place the N-queens on the squares of the board such that no two queens attack each other, meaning that no two queens are on the same row, the same column or the same diagonal.

You are given a configuration of N-queens on the board, which may not constitute a correct solution to the N-queens problem. Your goal is to transform the configuration into a correct solution by rearranging the queens in the fewest valid moves. In chess, the queen can move horizontally, vertically, or diagonally, and no other piece can occur between the starting square and the ending square of the move.

Solution 1

SAT Formula and Minizinc

Step 1: Find a satisfied assignment of N-queens on the board

To find a satisfying assignment to the n-queens formula, the formula needs to be defined. We can define two array to describe the assignment. One is target row R of size N: R[i] means there is a queen in row R[i]. The other one is target column C of size N: C[i] represents there is a queen in column C[i]. R[i], C[i] means there is a queen in position (R[i], C[i]).

  1. No two queens can share a row. As there are N queens, N rows and they cannot be at same row, each of the queen has one row. We set R = [1, 2, 3, … , N].

    Φ1 = ∀i(Ri = i)

  2. No two queens can share a column

    Φ2 = ∀i∀j(Ci ≠ Cj)

  3. No two queens can share a diagonal line

    Φ3 = ∀i∀j∀m(|R[i+m] - R[i]| ≠ |C[i+m] - C[i]|)

  4. Formula for n queens on the board:

    Φ = Φ1 ∧ Φ2 ∧ Φ3

Step 2: Count the moves from start configuration to target configuration

If queen x can move to target configuration directly, then move = move +1; else then move = move + 2.
For the queen x can move to the target position directly:

Φ5(x) = ∃i(R[x] = r[i] ∨ C[x] = c[i] ∨ (|R[x] - r[i]| = |C[x] - c[i]|))

Step 3: Find Minimum moves

Use Minizinc to find minimum moves by "solve minimize moves;"

Solution 2

Backtracking

The chess board can be seen as a graph. If two queens can attack each other, then add an edge between them. Therefore, we can find sets of independent set of the graph, where each independent set has size k. Suppose we take an independent set with size k at beginning, where k is the largest size among these independent sets. If k = n, then the queens are safe, and the move = 0. Else if k < n, we don't need to move the queens in the independent set as they won't attack each other. Then there are n - k queens need to move, the move amount is at least n – k (moves > n - k). If we found moves = n - k exists, then the moves is guaranteed to be the minimum move. Otherwise, we find if moves = n-k+1 exists.

Overall Pseudocode
Given: AllQueenSet, Pos, board_size; Independent Sets ≡Indpendent Set Algo(AllQueenSet,Pos,board_size)
k = FindMaxSizeOf(Independent Set);
if k == board_size
	then min move = 0, return min move
endif
while(k > 0)
	Sets_K = FindSetOfSizeK(Independent Sets, k)
	While Sets_K is not empty
		k_set = Sets_K.front()
		k_move = BackTrackAlgorithm(k_set, AllQueenSet, Pos, board_size)
		if found k_move and k_move == board_size – k // least move
			min move= k_move, return min move.
			Else if found k_move and k_move < min move
			Min move = k_move
		Endif
		Sets_K.pop()
	end while
	if found k_move
		return min move
	endif
	k = k - 1
end while
if no move found for all Independent set
	min move = board_size
return min move

Given: FixedQueens, FreeQueens, Position, board_size
FreeQueens = FreeQueens – FixedQueens
Find a list of freeSpots that don’t intersect with any of the Fixed Quees;

Backtrack
Given: Queen, FixedQueens, FreeQueens, Pos, FreeSpots, board_size
Int min_move = board_size; // it cannot exceed
For each spot ∈FreeSpots
	Int move;
	If spot and the queen is connected
		Then move = 1
		Else move = 2
	Endif
	Temp_Pos = Pos

	Temp_FreeQueens = FreeQueens – {Queen}
	// update free spots

	Else if Temp_FreeQueens not empty and

	End if
